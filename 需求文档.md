nkFlow 需求文档
项目概述
项目名：InkFlow
定位：基于 GitHub Actions + AI Agent 的多平台技术文章自动发布系统
核心理念：Write Once, Publish Everywhere —— 专注写作，发布交给 Agent 编排
使用流程（用户视角）
1. 在 Obsidian 中写文章（Markdown 格式）2. 文章头部配置 Front Matter 指定发布目标3. Obsidian Git 插件自动 Push 到 GitHub4. GitHub Actions 检测到新文章/更新5. Agent 读取配置，调用对应平台 Skill6. 文章自动发布到各目标平台7. 发布结果通过通知渠道反馈
Front Matter 规范
---title: 文章标题description: 文章摘要（100字以内）tags: [后端, K8s, 微服务]cover: ./images/cover.png   # 可选封面图publish:  - juejin  - csdn  - githubpages  - wechat            # 可选，推送到公众号草稿status: published       # draft=不发布 / published=发布date: 2026-03-11---
核心模块设计
模块一：触发层（Trigger）
监听 GitHub Push 事件
识别新增/修改的 Markdown 文件
过滤 status: draft 的文章
提取 Front Matter 配置
模块二：预处理层（Preprocessor）
Markdown 图片本地路径 → 上传阿里云 OSS → 替换为公网 URL
代码块格式兼容性处理（各平台渲染差异）
文章长度/格式校验
生成各平台适配版本（部分平台有字数/格式限制）
模块三：Agent 编排层（Orchestrator）
读取 publish 字段，生成发布任务列表
并行或串行调度各平台 Skill
失败重试机制（最多3次）
发布结果收集与汇总
模块四：Platform Skills（各平台技能）
Skill	实现方式	优先级
GitHub Pages	Git Push 触发 Jekyll/Hugo 构建	P0
掘金	掘金开放 API / 现有 Skill 整合	P0
CSDN	API / 模拟发布	P1
微信公众号	微信公众号草稿 API	P1
知乎	知乎创作者 API	P2
模块五：通知层（Notifier）
发布成功：输出各平台文章链接
发布失败：错误原因 + 重试状态
通知渠道：GitHub Actions Summary / 企业微信 / 邮件（可配置）

分阶段实现计划
Phase 1（MVP）
✓ GitHub Actions 触发 + Front Matter 解析✓ GitHub Pages 自动部署（最简单，先跑通流程）✓ 掘金自动发布（整合现有 Skill）✓ 基础通知（Actions Summary）
Phase 2
✓ 图片自动上传 OSS✓ CSDN 发布✓ 失败重试机制✓ 微信公众号草稿推送
Phase 3
✓ 知乎发布✓ 文章更新同步（已发布文章修改后同步更新）✓ 发布数据统计（阅读量汇总展示）✓ AI 辅助（自动生成摘要/标签建议）
技术选型
技术	选择	原因
运行环境	GitHub Actions	免费，触发方便，无需服务器
开发语言	Python	生态丰富，脚本场景最合适
Front Matter 解析	python-frontmatter	成熟库
HTTP 请求	httpx	支持异步，比 requests 更现代
静态站点	Hugo 或 Jekyll	GitHub Pages 原生支持
图片存储	阿里云 OSS	你已有经验

Phase 1 的核心任务就三件事：Actions 触发 → 解析 Front Matter → 调用掘金 Skill 发布，跑通这个最小闭环，后面的扩展就是加积木。
